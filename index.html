Git e GitHub

O que é Git?

Git é um sistema de controle de versão criado por Linus Torvalds. Ele permite:

  • Rastrear mudanças no código-fonte.
  • Trabalhar em equipe sem sobrescrever arquivos.
  • Voltar para versões anteriores do projeto.

O que é GitHub?

GitHub é uma plataforma online que armazena repositórios Git. Com ela você pode:

  • Compartilhar código com o mundo ou com sua equipe.
  • Colaborar com outras pessoas via pull requests.
  • Acompanhar bugs, tarefas e histórico de mudanças.

Diferenças entre Git e GitHub

  • Git: Ferramenta local para controle de versão.
  • GitHub: Serviço online que hospeda projetos Git.
  • Você pode usar Git sem GitHub, mas não o contrário.

Principais comandos Linux para usar no Git Bash

  • ls – Lista arquivos da pasta
  • cd pasta – Entra em uma pasta
  • mkdir nome – Cria uma nova pasta
  • touch arquivo.txt – Cria um novo arquivo
  • clear – Limpa o terminal

Comandos Linux presentes no README.md

git init             # Inicia o Git no projeto

git add .            # Adiciona todos os arquivos

git commit -m "msg"  # Cria um ponto de salvamento

git push origin main # Envia para o GitHub

Como criar um repositório no GitHub

  1. Acesse github.com e faça login.
  2. Clique em "New repository".
  3. Escolha o nome, a visibilidade (público ou privado).
  4. Adicione um README se quiser.
  5. Clique em "Create repository".

Como clonar um projeto existente

No terminal Git Bash, digite:

git clone https://github.com/usuario/repositorio.git

Isso vai copiar o repositório para sua máquina local.

Como gerar uma versão com Git (commit)

Após fazer mudanças nos arquivos:

git add .
git commit -m "mensagem explicando a alteração"

Esses comandos salvam uma "versão" do seu projeto.

Como enviar para o GitHub (push)

Depois de fazer commit, envie para o GitHub com:

git push origin main

O código local será enviado para o repositório online.
